常用事件小结

发表于 2015-10-24   |  

JS作为一门事件驱动的语言，其事件的重要性可想而知。常见的事件及其相关用法有些繁杂，在此总结一下，以备后用。

鼠标事件

  • click 单击事件
  • contextmenu 右击事件
  • dblclick 双击事件
  • mousedown 按下事件
  • mouseup 抬起事件
  • mousemove 移动事件
  • mouseover 移入事件
  • mouseout 移出事件
  • mouseenter 移入事件
  • mouseleave 移出事件
  • mousewheel 滚轮事件
  • DOMMouseScroll 滚轮事件

问：mousemove和mouseover事件有什么区别？

答：mouseover事件在鼠标移入到元素上时会触发，在元素内移动不会再次触发，mousemove事件当鼠标在元素上每移动一小段距离都会触发一次。

问：mouseover/mouseenter、mouseout/mouseleave有什么区别？

答：mouseover&mouseout默认是冒泡的，当给父元素添加此事件时，子元素的移入移出会再次触发。mouseenter&mouseleave是阻止冒泡的，一般来说后者更加符合我们的需求。

问：如何通过mousedown、mousemove、mouseup实现简单的拖拽功能？

答：大体上的思路是在mousedown时获得鼠标相对元素的位置，在mousemove时根据鼠标的偏移动态改变元素的位置，mouseup时取消move和up的事件。有三个关键点需要注意：

  1. 为了防止鼠标移动过快脱离元素造成拖动功能暂时失效，要将事件注册在document上。
  2. 为了防止父元素中有子元素时拖拽父元素，鼠标经过子元素会先触发子元素的mousemove造成混乱，需要将mousemove和mouseup都注册为捕获事件。
  3. 为了防止拖拽子元素时连同父元素也跟着走，需要阻止mousemove和mouseup的冒泡行为。
//不做兼容性处理
function drag(element) { 
    var deltaX, deltaY; //存储鼠标相对于元素的偏移位置。

    //注意这里，将事件注册在文档上。
    document.addEventListener('mousedown', downHandler);
    document.addEventListener('mousemove', moveHandler, true);
    document.addEventListener('mouseup', upHandler, true);

    function downHandler(e) {
        //这里假设元素绝对定位相对于body，否则需要精确计算其文档偏移量。
        //计算方法就是通过鼠标的文档坐标减去元素的文档坐标。
        deltaX = e.clientX + pageXOffset - element.offsetLeft;
        deltaY = e.clientY + pageYOffset - element.offsetTop;
    }

    function moveHandler(e) {
        element.style.left = e.clientX + pageXOffset - deltaX + 'px';
        element.style.top = e.clientY + pageYOffset - deltaY + 'px';
        e.stopPropagation();
    }

    function upHandler(e) {
        document.removeEventListener('mousemove', moveHandler, true);
        document.removeEventListener('mouseup', upHandler, true);
        e.stopPropagation();
    }
}

问：鼠标滚轮事件为什么有两个，它们的用法有什么不同？

答：一般滚轮事件是mousewheel，仅有firefox不支持它，所以只能用非标准的DOMMouseScroll来做兼容。在使用上也有区别，mousewheel事件对象上的wheelDelta能反映出滚动滚轮的情况，远离用户方向的一次滚动最小单位为120,接近的则为-120。

而对于DOMMouseScroll来说，利用事件对象上的detail属性反映滚轮情况。远离用户方向的一次滚动最小单位为-3,接近的则为3。

键盘与文本事件

  • keydown 按下事件
  • keyup 抬起事件
  • keypress 敲击事件
  • input 文本输入事件

问：input和key系列事件的本质区别是什么？

答：input事件上新输入的文本真正插入到聚焦的文档之后才触发，而key系列上插入到文档前触发。所以阻止keypress事件可以阻止用户的输入，input事件则可以用一些内容改变(全转换成大写)的场合。

问：keydown和keypress有什么区别？

答：可以简单地理解为keypress=keydown+keyup，但也并不完全是。比如一直按键不放时，keydown和keypress会交替连续触发。

另外，keypress常常不能显示组合键，一般在keypress不能捕获的时候使用keydown，keydown里面还有charCode属性，不过有兼容问题。
